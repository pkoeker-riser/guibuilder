Installationsanleitung GuiBuilder

Kurz und knapp für ganz eilige

1. Java installieren (falls noch nicht installiert)

  • Java Downloaden: http://java.sun.com/j2se/1.4/download.html (website)
    Hier runterscrollen bis "Download J2SE v 1.4.2" und z.B. klicken auf "Windows Offline Installation / SDK / Download"

2. GuiBuilder installieren

  • GuiBuilder.zip auspacken in irgendein Verzeichnis, am einfachsten nach C:\

3. GuiBuilder starten

  • C:\guibuilder\guibuilder.bat doppelklicken.

Fertig!

Für Fortgeschrittene und Profies

Installation Java

Voraussetzung für den Einsatz der GuiBuilders ist, daß das JDK ("Java Development Kit"; also die komplette Entwicklungsumgebung) oder JRE ("Java Runtime Environment"; also nur die Laufzeitumgebung) von Javasoft installiert ist.

Beides kann von www.javasoft.com lizenzfrei heruntergeladen werden.
Es ist jedoch zu beachten, daß im JRE kein Appletviewer (s.u.) enthalten ist.

Zur Zeit ist JDK 1.4 die aktuelle Version.

Siehe unter: http://java.sun.com/j2se/1.4/download.html (website)

Das JDK wird per Default unter "c:\j2sdk1.4" installiert.

JRE üblicherweise unter "c:\Programme\Java\j2re1.4.2".

Das Java Plug-in für Netscape und MSIE wird automatisch mit installiert. Opera bring ab Version 4.0 sein eigenes Plug-in mit; ältere Versionen können das Plug-in des Netscape mit benutzen.

Installation GuiBuilder

Application

Im einfachsten Fall setzen Sie den GuiBuilder nur als Application (also nicht als Applet) ein. Dann kann die ZIP-Datei guibuilder.zip in ein beliebiges Verzeichnis ausgepackt werden; Beispiel: C:\GuiBuilder.

XML-Bibliothek

Der Guibuilder funktioniert nur, wenn das Archiv GLUE.jar von TheMindElectric im Zugriff ist. Dieses wird im Verzeichnis /lib/ mit ausgeliefert;

Programmstart

Am einfachsten doppelklicken Sie die Datei GuiBuilder.bat; Sie können sich auch eine Verknüpfung dieser Datei auf den Desktop legen.

Hinweis für Fortgeschrittene:
Das Programm ist ein "Zwitter"; d.h. es kann sowohl als Application als auch als Applet eingesetzt werden.

Application

Im Verzeichnis guibuilder die BAT-Datei GuiBuilder.bat starten oder guibuilder.jar doppelklicken; letzteres bedeutet allerdings, daß keine DOS-Box mit etwaigen Fehler- oder Debug-Meldungen geöffnet wird.

Applet

Appletviewer

Vom DOS-Prompt aus dem Verzeichnis C:\guibuilder den Befehl appletviewer GApplet.html eingeben.

Fehlermeldungen wie "command not found" oder "applet not initialised" deuten darauf hin, daß etwas mit den Umgebungsvariablen nicht stimmt.

Browser

Der GuiBuilder läuft mit den Browsern, bei denen das Java Plug-in installiert ist: Netscape und IE.
Opera verwendet (bis Version 3.62) das Plug-in des Netscape mit, wenn in "opera.ini" das dort eingetragene Plug-in-Verzeichnis auf des entsprechende Verzeichnis des Netscape zeigt.

Unter "Start/Programme" (bzw. in der Systemsteuerung bei NT und Windows2000) findet sich in Folge der Installation des Java Plug-in ein Eintrag "Java Plug-in Control Panel".
Hier sollte für etwaige Fehlermeldungen das Kontrollkästchen "Show Java Console" aktiviert werden.

Der Eintrage unter "Java Runtime Environment" sollte auf das jeweilige JDK oder JRE gesetzt werden oder auf "Use Java Plug-in Default".

Für den Einsatz des Java Plug-in müssen spezielle HTML-Dateien verwendet werden. Sie sind darin zu erkennen, daß sie einen Tag <OBJECT> enthalten. Als Beispiel dient auch hier "GApplet.html". Das Applet kann aus dem Browser über "File / Open Page ..." gestartet werden.

Einsatz im Web

Das Werkzeug kann auch auf Basis eine Web-Servers eingesetzt werden. Installieren Sie dann den GuiBuilder in einem Verzeichnis, auf das der Webserver Zugriff hat und richten Sie im Browser eine Bookmark auf GApplet.html ein.

Das Problem besteht nur darin, daß das Applet Zugriff auf Ihre Spezifikationsdateien haben muß, welches üblicherweise im Web nur lesend, aber nicht schreibend möglich ist. Immerhin reicht diese Funktionalität aus, um eine fertige Spezifikation ablaufen zu lassen.

Wenn hingegen auch Daten aus dem Applet heraus geschrieben werden sollen, gibt es für die Dateiablage zwei Möglichkeiten:

  1. lokale Platte
  2. Webserver
1. Lokale Platte

Um dem Applet den Zugriff auf die lokale Platte zu erlauben, müssen Sie die Datei .java.policy im Verzeichnis /lib des GuiBuilder in Ihr "Home-"Directoy des Rechners kopieren. Dieses ist unter

  • Windows 95: c:/windows
  • Windows 95b : c:/windows/profiles/<username>
  • Windows NT:  c:/winnt/profiles/<username>
  • Windows 2000: c:/Dokument und Einstellungen/<username>

Achtung:
Sollte diese Datei dort bereits existieren, dürfen Sie diese nicht überschreiben, sondern den Inhalt mit einem normalen Texteditor zusammenfügen.

Diese policy-Datei erlaubt nun dem Applet auf das Verzeichnis c:/temp und seine Unterverzeichnisse zuzugreifen. Der Zugriff auf alle anderen Verzeichnisse ist dem Applet hingegen verwehrt (unter Unix muß dieser Eintrag naturgemäß in /temp geändert werden).

2. Dateiablage auf dem Webserver

Der Reiz dieser Lösung besteht darin, daß auf dem Client nur ein Browser mit dem Java Plug-in installiert werden muß, nicht aber der GuiBuilder selbst und alle Mitarbeiter im Projekt auf einer gemeinsamen Datenbasis arbeiten.

Hier besteht die Einschränkung zumeist darin, daß Web-Server üblicherweise so eingerichtet sind, daß sie keine HTTP-PUT zulassen, oder über diese Funktionalität garnicht verfügen.

Außerdem muß sich ein Benutzer üblicherweise mit Username und Password authentifizieren. Der Administator des Webservers muß die berechtigten User zuvor einrichten (Siehe bei Sambar unter "Server Configuration" / "HTTP-PUT Users"). Trotzdem sollten Sie diese Konfiguration nur im gesicherten Intranet, nie aber im allgemein zugänglichen Internet einrichten.

Dem HTML-Files, welches das Applet der GuiBuilders aufruft, muß ein Parameter für die Verwendung von HTTP beigegeben werden; hier sind die Werte "NONE", "GET" und "PUT" erlaubt.

<param name="HttpUsage" value="PUT">

Als Beispiel dient auch hier das Applet "GApplet.html".

Diese Konfiguration wurde getestet mit

  • Mozilla 1.5
  • Netscape 7.1
  • Opera 6.04
  • MSIE 5.0 / 6.0
  • und Sambar 5.3. (website)
3. FTP-Client

Mit einem handelsüblichen FTP-Client (wie z.B. WS_FTP) können die Spezifikationen zwischen Client und Server ausgetauscht werden.

Startfenster an GuiBuilder übergeben

Soll im Rahmen einer Präsentation die mit dem GuiBuilder erstellte "Anwendung" demonstriert werden, kann ihm als Argument das als erstes zu startende Script übergeben werden:

  • Application
    Erstellen Sie eine BAT-Datei, die den GuiBuilder mit einem zusätzlichen Argument aufruft; Beispiel:
    java -jar guibuilder.jar Login.xml
    Einen Verweis dieser BAT-Datei können Sie dann ins Startmenü von Windows einbinden oder auf dem Desktop ablegen.
    Zusätzlich können weitere Parameter mit Leerzeichen getrennt gesetzt werden:
    StartWindow=<StartWindow>
    DocumentBase=<docbase>
    UiManager=Windows|Metal|Motif|GTK
    Version=<version>
  • Applet
    In die HTML-Datei des Applet muß ein Paramter "STARTWINDOW" auf den gewünschten Wert gesetzt werden; Beispiel:
    <PARAM NAME = STARTWINDOW VALUE = "Login.xml">
    Hier können auch weitere Parameter gesetzt werden: DOCUMENTBASE für das Verzeichnis, aus dem die Oberflächenscripte geladen werden sollen; UIMANAGER und VERSION.
